커스텀 엘리먼트란?

커스텀 엘리먼트(Custom Elements)는 HTML 표준에서 제공하는 기능으로, 개발자가 직접 새로운 HTML 태그(컴포넌트)를 만들 수 있게 해주는 기술입니다.

예시: <site-modal>, <site-scroll-reveal> 등 기존에 없던 태그를 직접 정의해서 사용할 수 있습니다.

왜 이 프로젝트에서 커스텀 엘리먼트를 쓰나요?

  • 재사용성: 한 번 정의하면 어디서든 같은 방식으로 사용 가능
  • 캡슐화: 내부 동작/스타일이 외부와 분리되어 충돌 위험이 적음
  • HTML만으로 선언적 사용: 태그만 선언하면 동작, 별도 JS 조작 없이도 사용 가능
  • 프레임워크 독립적: 어떤 환경에서도 사용 가능
  • 유지보수 효율화: 기능 수정/추가가 한 곳에서만 이뤄짐

이 프로젝트는 브랜드별로 디자인이 달라도, 반복적으로 사용되는 핵심 기능(모달, 토스트, 스크롤 효과 등)을 쉽게 재사용할 수 있도록 커스텀 엘리먼트 방식을 채택했습니다.

커스텀 엘리먼트 문법 설명

  • 클래스 정의: 커스텀 엘리먼트는 HTMLElement를 상속받아 클래스로 정의합니다.
  • 라이프사이클 메서드:
    • connectedCallback(): 요소가 DOM에 추가될 때 자동 실행
    • disconnectedCallback(): 요소가 DOM에서 제거될 때 자동 실행
    • attributeChangedCallback(name, oldVal, newVal): 지정한 속성이 변경될 때 실행
    • static get observedAttributes(): 감시할 속성명 배열 반환 (attributeChangedCallback과 함께 사용)
  • 속성/메서드 활용: this.getAttribute('속성명')으로 속성값을 읽거나, 클래스 내부에 메서드를 정의해 외부 JS에서 호출할 수 있습니다.
  • 내부 렌더링: this.innerHTML 또는 this.appendChild() 등으로 내부 구조를 동적으로 생성할 수 있습니다.
  • 재사용성: 한 번 정의하면 HTML 어디서든 태그만으로 사용할 수 있습니다.
  • 프레임워크 독립적: jQuery, React, Vue 등 어떤 환경에서도 동작합니다.
  • 등록: customElements.define('태그명', 클래스명)으로 HTML에서 사용할 수 있도록 등록합니다. 태그명에는 반드시 하이픈(-)이 포함되어야 합니다.

이러한 문법 구조 덕분에 커스텀 엘리먼트는 재사용성, 유지보수성, 확장성이 뛰어난 UI 컴포넌트 개발이 가능합니다.

커스텀 엘리먼트 구조 예시


// 커스텀 엘리먼트의 전체 구조와 주요 메서드 예시
class MyElement extends HTMLElement {
  constructor() {
    super();
    // [필수] 인스턴스가 생성될 때(딱 1번) 호출됨. 초기화 코드 작성
  }

  connectedCallback() {
    // [필수] 요소가 DOM에 추가될 때마다 호출됨 (여러 번 가능)
  }

  disconnectedCallback() {
    // 요소가 DOM에서 제거될 때마다 호출됨 (여러 번 가능)
  }

  static get observedAttributes() {
    // (필요시) 감시할 속성명 배열 반환 (attributeChangedCallback과 함께 사용)
    return ['attr1', 'attr2'];
  }

  attributeChangedCallback(name, oldValue, newValue) {
    // (필요시) observedAttributes에 명시된 속성 변경 시마다 호출됨
  }

  adoptedCallback() {
    // (고급) 요소가 다른 document로 이동될 때 호출됨 (거의 사용되지 않음)
  }

  // 기타 커스텀 메서드 및 속성 정의 가능
  myCustomMethod() {
    // 개발자가 원하는 추가 메서드
  }
}

// [필수] 커스텀 엘리먼트 등록 (태그명에는 반드시 하이픈(-) 포함)
customElements.define('my-element', MyElement);

jQuery 방식과의 차이

jQuery 방식 커스텀 엘리먼트 방식
DOM 조작/이벤트 바인딩을 직접 구현 태그만 선언하면 동작(내부에 코드 내장)
코드가 여러 곳에 분산 기능별로 한 곳에 집중
스타일/동작 충돌 위험 캡슐화로 충돌 최소화
재사용/유지보수 어려움 재사용/유지보수 용이

🎨 브랜드별 커스터마이징

site-ui.js는 다양한 브랜드에서 활용할 수 있도록 강력한 커스터마이징 시스템을 제공합니다.

1. 훅(Hook) 시스템 활용

컴포넌트의 생명주기 각 단계에서 커스텀 로직을 실행할 수 있습니다.

// 브랜드 A: 슬라이드 변경 시 GA 이벤트 전송
const swiper = document.querySelector('site-swiper');

swiper.addHook('slideChangeEnd', function(swiper) {
  // Google Analytics 이벤트 전송
  gtag('event', 'slide_change', {
    slide_index: swiper.activeIndex,
    brand: 'brand-a'
  });
});

// 브랜드 B: 슬라이드 변경 시 사운드 효과
swiper.addHook('slideChangeStart', function() {
  const audio = new Audio('/sounds/slide-whoosh.mp3');
  audio.volume = 0.3;
  audio.play();
});

사용 가능한 훅 목록

  • beforeConnect - 컴포넌트가 DOM에 연결되기 전
  • afterConnect - 컴포넌트가 DOM에 연결된 후
  • beforeInit - Swiper 초기화 전
  • afterInit - Swiper 초기화 후
  • configReady - 설정이 준비된 후
  • slideChangeStart - 슬라이드 변경 시작
  • slideChangeEnd - 슬라이드 변경 완료
  • beforeRender - HTML 구조 렌더링 전
  • afterRender - HTML 구조 렌더링 후

2. 브랜드별 클래스 상속

기본 컴포넌트를 상속하여 브랜드별 특화 컴포넌트를 만들 수 있습니다.

// 브랜드 A 전용 Swiper
class BrandASwiper extends SiteSwiper {
  getDefaultConfig() {
    const config = super.getDefaultConfig();
    // 브랜드 A 기본 설정
    return {
      ...config,
      speed: 800, // 더 느린 전환
      effect: 'fade', // 페이드 효과 기본값
      autoplay: {
        delay: 5000,
        disableOnInteraction: false
      }
    };
  }

  connectedCallback() {
    // 브랜드 A 전용 훅 자동 등록
    this.addHook('afterInit', (swiper) => {
      // 브랜드 A 로고 워터마크 추가
      const watermark = document.createElement('div');
      watermark.className = 'brand-a-watermark';
      watermark.innerHTML = '<img src="/brand-a-logo.png" alt="Brand A">';
      this.appendChild(watermark);
    });
    
    super.connectedCallback();
  }
}

// 커스텀 엘리먼트 등록
customElements.define('brand-a-swiper', BrandASwiper);

3. 동적 컨텐츠 관리

런타임에 슬라이드를 추가/제거하거나 설정을 변경할 수 있습니다.

const swiper = document.querySelector('site-swiper');

// 동적 슬라이드 추가
swiper.addSlide(`
  <div class="slide-content">
    <h3>새로운 슬라이드</h3>
    <p>동적으로 추가된 컨텐츠입니다.</p>
  </div>
`);

// 특정 위치에 슬라이드 삽입
swiper.addSlide('<div>첫 번째 슬라이드</div>', 0);

// 슬라이드 제거
swiper.removeSlide(2);

// Swiper 인스턴스 직접 접근
const swiperInstance = swiper.getSwiperInstance();
swiperInstance.slideTo(3); // 3번째 슬라이드로 이동

4. 이벤트 기반 통신

컴포넌트는 표준 DOM 이벤트를 발생시켜 다른 시스템과 통신할 수 있습니다.

const swiper = document.querySelector('site-swiper');

// Swiper 준비 완료 이벤트 리스닝
swiper.addEventListener('swiperReady', (event) => {
  const { swiper, config } = event.detail;
  console.log('Swiper 초기화 완료!', swiper, config);
  
  // 다른 컴포넌트에 알림
  document.dispatchEvent(new CustomEvent('heroSwiperReady', {
    detail: { totalSlides: swiper.slides.length }
  }));
});

5. 브랜드 적용 예시

브랜드 커스터마이징 예시입니다.

// brand-custom.js
(function() {
  'use strict';
  
  // 브랜드별 설정
  const BRAND_CONFIG = {
    name: 'premium-brand',
    colors: {
      primary: '#d4af37',
      secondary: '#1a1a1a'
    },
    animations: {
      speed: 1200,
      easing: 'cubic-bezier(0.4, 0, 0.2, 1)'
    }
  };

  // 페이지 로드 후 모든 Swiper에 브랜드 설정 적용
  document.addEventListener('DOMContentLoaded', function() {
    const swipers = document.querySelectorAll('site-swiper');
    
    swipers.forEach(swiper => {
      // 브랜드 클래스 추가
      swiper.classList.add(BRAND_CONFIG.name);
      
      // 고급 애니메이션 훅
      swiper.addHook('slideChangeStart', function(swiperInstance) {
        // 이전 슬라이드 페이드아웃
        const prevSlide = swiperInstance.slides[swiperInstance.previousIndex];
        if (prevSlide) {
          prevSlide.style.transition = `opacity ${BRAND_CONFIG.animations.speed}ms ${BRAND_CONFIG.animations.easing}`;
          prevSlide.style.opacity = '0.7';
        }
        
        // 현재 슬라이드 강조
        const activeSlide = swiperInstance.slides[swiperInstance.activeIndex];
        activeSlide.style.opacity = '1';
      });
      
      // 분석 데이터 수집
      swiper.addHook('slideChangeEnd', function(swiperInstance) {
        // 브랜드별 분석 시스템에 데이터 전송
        if (window.brandAnalytics) {
          window.brandAnalytics.track('swiper_interaction', {
            slide_index: swiperInstance.activeIndex,
            total_slides: swiperInstance.slides.length,
            brand: BRAND_CONFIG.name
          });
        }
      });
    });
  });
})();
